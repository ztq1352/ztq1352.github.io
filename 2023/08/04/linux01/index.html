linux入基础

概述

linux指的是内核，内核基础上制作成发行版：Ubuntu(乌班图)、RedHat(红帽)、CentOS等

应用场景

linux运维工程师、linux嵌入式工程师

开发项目：JavaEE、大数据、Python、PHP、C/C++、go

应用领域

个人桌面领域、服务器领域、嵌入式领域

VM的使用

分区

boot（1G）：标准分区，ext4

swap（2G）： 标准分区，swap

tool（剩余17G）：标准分区，ext4

网络连接三种模式

桥接模式

虚拟机可以和外部系统通讯，但是容易造成IP冲突。

NAT模式

网络地址转换模式，虚拟机可以和外部系统通讯，不造成IP冲突。但是虚拟系统IP无法直接与其他外界主机联系，需要通过生成的代理IP联系。

主机模式

独立的系统，不和外界通讯。

虚拟机linux克隆

方式1：直接拷贝一份安装好的虚拟机文件，用VM打开就行。

方式2：使用VM克隆操作，克隆时需要关闭linux系统。采用完整克隆才是新的linux系统。

虚拟机的迁移和删除

可将虚拟机文件直接拷贝或剪切到其他位置，用VM打开即可。删除则可用VM进行移除(实际文件未删除)或在文件夹中直接删除虚拟系统对应的文件。

虚拟机快照

担心接下来的操作可能造成系统异常，需要回到某个正常运行的状态，使用VM进行快照保存。

VMtools

可以在windows下更好的管理vm虚拟机

可以设置windows和centos的共享文件夹

安装步骤

VM选择安装VMware Tools，linux显示VMtools安装驱动

进入主文件夹，找到VMtools驱动，找到压缩文件，复制到opt文件中，进入该文件夹终端（cd /opt）

输入tar -zxvf xx.tar.gz 进行解压

进入解压文件夹 cd v….

输入 ./vmware-install.pl 安装，全程回车

注意：安装vmtools，需要gcc，查看是否安装用gcc -v

Linux实用操作

Linux目录结构

层级式树状目录结构，最上层是 “/“ 根目录。

第二层：

  • /bin：存放最经常使用的命令
  • /sbin：系统管理员使用的管理程序
  • /home ：存放普通用户的主目录，一般在Linux中每个用户都有自己的目录。
  • /root：系统管理员，也称超级权限者的用户主目录。
  • /lib：系统开机所需最基本的动态连接共享库，类似windows里的DLL文件，几乎所有的应用程序都需要这些共享库。
  • /lost+found:一般情况是空的，当系统非法关机后，会有存放文件。
  • /etc：所有系统管理所需要的配置文件和子目录，比如安装的mysql的配置文件也放入当中。
  • /usr：非常重要的目录，用户很多应用程序的文件都放在这个目录下，类似windows下的program files目录。
  • /boot：存放的是启动Linux时使用的一些核心文件，包括一些连接文件以及镜像文件。
  • /proc：虚拟目录。是系统内存的映射，访问这个目录来获取系统信息。
  • /srv：service缩写，该目录存放一些服务启动之后需要提取的数据。
  • /sys：该目录安装了2.6内核新出现的一个文件系统sysfs。
  • /temp：存放临时文件。
  • /dev：类似windows的设备管理器，把所有的硬件用文件的形式存储。
  • /media：linux系统会自动识别一些设备，例如U盘、光驱等，当识别后，linux会把别识别的设备挂载在这个目录下。
  • /mnt：为了让用户临时挂载别的文件系统的，可以将外部存储挂载在/mnt上，进入目录即可查看内容。
  • /opt：这是给主机额外安装软件所摆放的目录。如安装oracle数据库即可放置该目录，默认为空。
  • /usr/local：这是另一个主机额外安装软件所安装的目录，一般是通过编译源码方式安装的程序。
  • /var：该目录存放不断扩充的东西，习惯将经常被修改的目录放置在该目录下，包括各种日志文件等。
  • /selinux：是一种安全子系统，它能控制程序只能访问特定文件，有三种工作方式，可自行设置。

vi和vim编辑器

常用的三种模式：

  • 正常模式（默认）：可以使用“上下左右”按键移动光标，“删除字符”或“删除整行”来处理文档内容，也可“复制粘贴”处理文件数据。
  • 插入模式：按下“i、I、o、O、a、A、r、R等”任何一个字母之后进入编辑模式，一般用i即可。
  • 命令行模式：提供完成读取、存盘、替换、离开vim、显示行号等动作。

各种模式切换

  1. vim xxx 进入文件
  2. i或a进入编辑模式，按Esc退出编辑模式
  3. 输入：或/进入命令行模式，按Esc退出命令行模式，：wq（保存退出）、：q（退出）、：q！（强制退出不保存）

常用编辑快捷键（区分大小写）

  1. 拷贝当前行“yy”，拷贝当前行下的n行，“nyy”，并粘贴（输入p）

  2. 删除当前行“dd”，拷删除当前行下的n行，“ndd”

  3. 在文件中查找某个单词，命令行/+关键字，回车查找，输入n查找下一个

  4. 设置文件中的行号“：set nu”，取消文件中的行号“：set nonu”

  5. 编辑某个文件，文件内容很多行，在一般模式下，“G”定位到文档的最末行，“gg”定位到最首行

  6. 在文件中输入字符，在一般模式下，撤销用“u”

  7. 编辑文件中，在一般模式下，光标定位到某行，输入 n行+shift+g

  8. …..

linux关机&重启命令

shutdown -h now 立刻进行关机

shutdown -h 1 一分钟后关机

shutdown -r now 现在重启

halt 关机

reboot 重启

sync 把内存的数据同步到磁盘中

注意细节：不管是重启还是关机，首先要运行sync命令，再进行关机或重启，小心使得万年船。

用户登录和注销

切换用户：su - 用户名，后输入密码（权限高的用户切换到权限低的用户不需要输入密码，反之需要，返回原来用户用logout/exit指令）

注销用户：logout，注意：在图形运行级别无效，在运行级别3下有效。

用户管理

添加用户：useradd 用户名（在/home目录中会添加该用户文件夹）

添加用户时指定目录：useradd -d 目录地址 新用户名字

设置密码：passwd 用户名（用户名一定要写，不然修改当前用户的密码）

删除用户：保留家目录“userdel 用户名”，不保留家目录“userdel -r 用户名”，一般情况下建议保留家目录。

查看第一次登录的用户：who am i

查看当前登录的用户：whoami

查看当前用户所在的目录：pwd

查看当前目录下的所有文件名：ls

用户组

新增组：groupadd 组名

删除组：groupdel 组名

增加用户时直接添加组：useradd -g 用户组 用户名

修改用户的组：usermod -g 用户组 用户名

  • /etc/passwd 文件

记录用户的各种信息

每行含义：用户名:口令:用户标识号:注释性描述:主目录:登录Shell

  • /etc/shadow文件

口令的配置文件

每行含义：登录名:加密口令:最后一次修改时间:最小时间间隔:最大时间间隔:警告时间:不活动时间:失效时间:标志

  • /etc/group 文件

组的配置文件，记录Linux包含的组的信息

每行含义：组名:口令:组标识号:组内用户列表

Linux中每个用户必须属于一个组，不能独立于组外

权限基本介绍

ll指令内容：drwxr-xr-x. 3 root root 4096 7月 31 17:21 zeng

drwxr-xr-x.：3 ：root： root： 4096： 7月 31 17:21： zeng

文件类型权限： 文件硬链接或子目录数： 所属用户名： 所属用户组： 文件大小： 最后修改日期： 文件名

0~9位说明：

第0位：

确定文档类型（d、-、l、c、b）

l：链接，相当于windows快捷方式

d：目录，相当于windows文件夹

c：块设备，，比如硬盘等

b：字符设备，比如鼠标、键盘等

第1~3位：

确定该文件所有者拥有该文件的权限–User

第4~6位：

确定所属组（同用户组的）拥有该文件的权限–Group

第7~9位：

确定其他用户拥有该文件的权限–Other

rwx作用到文件

r：代表可读(read) 可以读取查看

w：代表可写(write) 可以修改，但是不代表可以删除该文件，删除一个文件的前提条件是对该文件所在的目录有写权限，才能删除该文件。

x：代表可执行(execute) 可以被执行

rwx作用到目录

r：代表可读(read) 可以读取，ls查看目录内容

w：代表可写(write) 可以修改，对目录内创建+删除+重命名该目录

x：代表可执行(execute) 可以进入该目录

修改权限-chmod

通过chmod指令，可以修改文件或者目录的权限

第一种方式：+、-、=

u：所有者、g：所有组：o：其他人、a：所有人(u、g、o的总和)

chmod u=rwx，g=rx，o=x 文件/目录名

chmod o+w 文件/目录名

chmod a-x 文件/目录名

第二种方式：数字

r=4，w=2，x=1 rwx = 1+2+4 =7

chmod u=rwx，g=rx，o=x 文件/目录名

相当于 chmod 751 文件/目录名

修改文件所有者-chown

改变所有者：chown 新的所有者 文件/目录

改变所有者和所在组：chown 新的所有者:新的组 文件/目录

-R：如果是目录，则使其下所有子目录或文件递归生效

修改文件所在组-chgrp

改变所在组：chgrp 新的组名 文件/目录

-R：如果是目录，则使其下所有子目录或文件递归生效

Linux定时任务调度

crond调度

任务调度：是指系统在某个时间执行的特定的命令或程序

任务调度分类：

1.系统工作：有些重要的工作必须周而复始的执行，如病毒扫描等

2.个别用户工作：个别用户可能希望执行某些程序，比如对mysql数据库备份等

基本语法：

crontab 【选项】

其他指令：service crond restart 重启任务调度

常用选项：

-e 编辑crontab定时任务

-l查询crontab任务

-r删除当前用户所有的crontab任务

快速入门

设置任务调度文件：/etc/crontab

设置个人任务调度，执行crontab -e命令

接着输入任务到调度文件中：*/1 * * * * ls -| /etc/ > /tmp/to.txt

即每小时的每分钟执行ls -| /etc/ > /tmp/to.txt命令

占位符说明

第一个*：一小时当中的第几分钟（0~59）

第二个*：一天当中的第几小时（0~23）

第三个*：一个月当中的第几天（1~31）

第四个*：一年当中的第几个月（1~12）

第五个*：一周当中的星期几（0~7，0和7都是周日）

特殊符号说明

*：代表任何时间

，：代表不连续的时间

‘-’：代表连续的范围时间

*/m：代表每隔多久执行一次

at定时任务

1.at指令是一次性定时计划任务，at的守护进程atd会以后台模式运行，检查作业队列来运行。

2.默认情况下，atd守护进程每60秒检查作业队列，有作业时，会检查作业运行时间，如果时间与当前时间匹配，则运行此作业。

3.at命令时一次性定时计划任务，执行完一个任务后不再执行此任务了

4.在使用at命令的时候，一定要保证atd进程的启动，可以用相关指令查看：ps -ef | grep atd

at命令格式

at 【选项】【时间】

at命令选项：

-m：当指定的任务完成后，将给用户发送邮件，即使没有标准输出

-I：atq的别名

-d：atrm的别名

-V：显示任务被执行的时间

-q<队列>：使用指定的队列

-f<文件>：从指定文件读入任务而不是从标准输入读入

-t<时间参数>：以时间参数的形式提交要运行的任务

Ctrl + D 结束at命令的输入

Linux磁盘分区、挂载

Linux分区

1.无论有几个分区，分给哪一个目录，它归根结底只有一个目录，一个独立且唯一的文件结构，Linux中每一个分区都是用来组成整个文件系统的一部分。

2.Linux采用“载入”的处理方法，它的整个文件系统中包含了一整套的文件和目录，且将一个分区和一个目录联系起来。这时要载入一个分区将使它的存储空间在一个目录下获得。

硬盘说明

Linux硬盘分为IDE和SCSI硬盘，目前基本上是SCSI

对于IDE硬盘，驱动器标识号是“hdx”，SCSI硬盘标识号是“sdx”

其中“hd”是指分区所在的设备类型

“x”指盘号：a为基本盘，b为基本从属盘，c为辅助主盘，d为辅助从属盘

“”代表前四个分区，用数字14表示

查看所有设备挂载情况

命令：lsblk 或 lsblk -f

挂载经典案例

  1. 虚拟机创建硬盘（重启系统生效）

  2. 分区命令：fdisk /dev/sdb

输入m 显示命令列表

输入p显示磁盘分区

输入n新增分区

输入d删除分区

输入w写入并退出

操作说明：开始分区后输入n（新增分区），然后选择p（分区类型为主分区），两次回车默认剩余全部空间，最后输入w（写入并退出），若不保存退出输入q。

  1. 格式化磁盘

分区命令：mkfs -t ext4 /dev/sdb1

其中ext4是分区类型

  1. 挂载

将一个分区与一个目录联系起来

挂载命令：mount 设备名称 挂载目录

例如：mount /dev/sdb1 /newdisk

取消挂载命令：umount 设备名称或者挂载目录

例如：umount /dev/sdb1 或 umount /newdisk

注意：用命令行挂载，重启后会失效

  1. 永久挂载

修改/etc/fstab文件

例如：/dev/sdb1 /newdisk ext4 defaults 0 0

修改后执行mount -a生效

Linux进程管理

在LINUX中，每个执行的程序都称为一个进程。每一个进程都分配一个ID号(pid,进程号)

每个进程都可能以两种方式存在的。前台与后台，所谓前台进程就是用户目前的屏慕上可以进行操作的。后台进程则是实际在操作，但由于屏幕上无法看到的进程，通常使用后台方式执行。

一般系统的服务都是以后台进程的方式存在，而且都会常驻在系统中。直到关机才结束。

显示执行的进程-ps

  • 基本指令：ps 【选项】

常用方式：ps -aux | more 、ps -aux | grep xxx

ps -a：显示当前终端的所有进程信息

ps -u：以用户的格式显示进程信息

ps -x：显示后台进程运行的参数

  • 显示的信息解释

USER：进程执行 用户

PID：进程识别号

CPU：占用CPU的百分比

MEN：占用物理内存的百分比

VSZ：占用虚拟内存大小KB

RSS：占用物理内存大小KB

TTY：终端机号

STAT：运行状态，s休眠、r运行，n低优先级 ，d短期等待，z死僵进程，t被追踪或被停止 等

START：执行开始的时间

TIME：此进程所耗CPU时间

COMMAND：正在执行的命令或进程名

  • 以全格式显示当前所有进程，并查看进程

指令：ps -ef

-e：显示是所有进程

-f：全格式

PID : 进程ID
PPID : 父进程ID

注：通过PPID明白父进程

服务管理-service

服务(service) 本质就是进程如(mysgl,sshd 防火墙等)，但是是运行在后台的，通常都会监听某个端口，等待其它程序的请求
因此我们又称为守护进程

服务的运行级别

Linux系统有7种运行级别(runlevel) : 常用的是级别3和5

运行级别0 : 系统停机状态，系统默认运行级别不能设为0，否则不能正常启动

运行级别1 : 单用户工作状态，root权限，用于系统维护，禁止远程登陆

运行级别2 : 多用户状态(没有NFS)，不支持网络

运行级别3 : 完全的多用户状态(有NFS)，登陆后进入控制台命令行模式

运行级别4 : 系统未使用，保留

运行级别5 : X11控制台，登陆后进入图形GUI模式

运行级别6: 系统正常关闭并重启，默认运行级别不能设为6，否则不能正常启动

  • 开机流程

开机—》BIOS—》/boot—》systemlt进程1—》运行级别—》运行对应的级别服务

设置运行级别：

查看当前运行级别：systemctl get-default

3：systemctl set-default multi-user.target

5：systemctl set-default graphical.target

Linux 的rpm包管理

rpm查询指令

rpm用于互联网下载包的打包及安装工具，它包含在某些Linux分发版中。它生成具有.RPM扩展名的文件。RPM是RedHat Package Manager ( RedHat软件包管理工具)的缩写，类似windows的setup.exe，这一文件格式名称虽然打上了RedHat的标志，但理念是通用的。
Linux的分发版本都有采用( suse,redhat, centos 等等)，可以算是公认的行业标准了

  • rpm包的简单查询指令

    查询已安装的rpm列表 rpm -qa l grep xx

  • rpm包名基本格式

    一个rpm包名 : firefox-60.2.2-1.el7.centos.x86_64

    名称:firefox

    版本号: 60.2.2-1

    适用操作系统: el7.centos.x86_64

    表示centos7.x的64位系统

    如果是i686、i386表示32位系统，noarch表示通用

  • rpm包的其它查询指令

    rpm -qa ：查询所安装的所有rpm软件包

    rpm -qa | more

    rpm -qa I grep X [rpm -qa I grep firefox ]

    rpm -q 软件包名：查询软件包是否安装

    案例 : rpm -q firefox

    rpm -qi 软件包名：查询软件包信息

    案例: rpm -qi firefox

    rpm -ql 软件包名 ：查询软件包中的文件

    比如 : rpm -ql firefox

    rpm -qf 文件全路径名 查询文件所属的软件包

    rpm -qf /etc/passwd

安装JDK

安装步骤

  1. mkdir /opt/jdk 创建专门存储安装包的文件夹
  2. 通过xftp 上传到 /opt/jdk中
  3. cd /opt/jdk
  4. 解压 tar -zxvf jdk…..tar.gz
  5. mkdir /usr/local/java 创建专门存放安装的目录的文件夹
  6. mv /opt/jdk/jdk… /usr/local/java 移动解压的文件到存放的地方
  7. 配置环境变量：vim /etc/profile
  8. 在文件末尾添加：export JAVA_HOME=/usr/local/java/jdk….
  9. export PATH=$JAVA_HOME/bin:$PATH
  10. source /etc/profile 让新的环境变量生效
  11. 测试是否安装成功

在其他目录下输入：javac -version、java -version

也可编写一个Hello.java文件编译

安装mysql

  1. 创建/opt/mysql文件夹，并cd进入目录 mkdir /opt/mysql
  2. 将下载的mysql文件通过xftp传输到/opt/mysql目录
  3. 运行 tar -xvf mysql……tar 解压（不用z选项是因为包为tar文件）
  4. 运行rpm -qa |grep mari 查询mari相关包，并将mari相关包删除（因为CentOS自带的数据库mariadb，会跟mysql冲突）
  5. 运行rpm -e –nodeps mariadb.. 卸载
  6. 然后开始真正安装mysql，依次运行如下文件

rpm -ivh mysql-community-common….rpm

rpm -ivh mysql-community-libs….rpm

rpm -ivh mysql-community-client….rpm

rpm -ivh mysql-community-server…..rpm

  1. 安装完成后运行 systemctl start mysql.service 启动mysql
  2. 然后开始设置root用户密码

Mysql自动给root用户设置随机密码，运行 grep ”password“ /var/log/mysqld/log 可看到密码

  1. 运行mysql -u root -p 用root用户登录，提示输入密码，用上述查到的密码输入即可登录
  2. 设置root密码，对于个人开发环境，可设置简单的密码（生成环境需要复杂的)可运行set global validate_password_policy=0

密码设置策略默认是1的级别，

低：0 长度大于8位即可

中：1 长度大于8位且要求存在数字、大小写字母、特殊字符

高：2 长度大于8且要求数字、大小写字母、特殊字符、字典文件

  1. 设置密码 set password for ‘root‘@’localhost’=password(‘123456’);
  2. 运行flush privileges; 使密码设置生效
