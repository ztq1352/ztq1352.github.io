项目分析

  1. 项目功能

     登录、注册、热销商品、用户管理（密码、个人信息、头像、收货地址）
 购物车（展示、增加、删除）、订单模块

  2. 开发顺序

     注册、登录、用户管理、购物车、商品、订单模块

  3. 某个模块开发：

     持久层：依据前端页面的设置规划相关的SQL语句，以及进行配置
 业务层开发：核心功能控制、业务操作以及异常的处理
 控制层开发：接收请求、处理响应
 前端开发：js、jQuery、ajax

项目搭建

  1. 项目环境

     JDK：1.8以上
 Maven：配置到idea，3.6.1版本
 数据库：MySQL。要求5.1以上版本
 开发平台：idea开发

  2. 项目名称

     store:表示商城

  3. 结构

     com.cy.store

  4. 基础搭建工具选择

     Java web、mybatis、mysql driver、lombok，后续工具后面添加

  5. 资源文件

     resources文件下(static、templates)

  6. 单元测试

     路径：test.com.cy.store，在实际测试中使用分层测试

  7. 创建参数

     group：store
 artifact：com.cy
 java version：自己使用的版本

设计接口和抽象方法

定义mapper接口，在项目的目录结构创建mapper包，在这个包下根据不同的功能模块创建mapper接口，创建UserMapper接口。
映射文件名称与mapper接口名称保持一致

编写映射

定义xml映射文件，与对应的接口相关联，在resources目录下创建一个mapper文件夹存放mapper映射文件

测试

在测试包下创建mapper包，包下创建UserMapperTests类，添加@SpringBootTest和@RunWith(SpringRunner.class)注解
注入UserMapper，使用@Test注解测试方法

测试结果：insert()打印影响行数，观察数据库变化，select()打印对应数据

学习总结

在这次学习中学到如下内容：

  • 了解mybatis开发流程，规划sql语句–>设计接口和抽象方法–>编写映射–>测试。
  • 使用按模块单元测试方法。
  • 运用java继承知识，创建基类实现异常、实体类优化处理。
  • 使用JSON返回结果集的Restful风格。
  • MD5加密原理。
  • 持久层、业务层、控制层代码编写。
  • Ajax前后端交互。
